## Development Environment Setup
This should just be Markdown files, so you'll want an editor that works for MarkDown.

A variety of tools exist. I use VSCode (it's probably overkill for simple changes). A basic text editor like `Notepad` can work with Markdown. 

If you're new to Markdown, the [Markdown Guide](https://www.markdownguide.org/) may be helpful.


## Styleguide
To help keep content from multiple editors consistent, this styleguide is defined:

 * [Inclusive](https://developers.google.com/style/inclusive-documentation) and gender-neutral language; `you`, `they`, `them`, `theirs`
 * [Oxford spelling](https://en.wikipedia.org/wiki/Oxford_spelling) `en-GB-oxendict` (or `en-CA` (Candian) for broader support) 
    * e.g.
      * `-ize` ending (where gramatically correct)

    * Please be consistent with existing spellings
 * [Oxford commas](https://en.wikipedia.org/wiki/Serial_comma)
 * List items shouldn't have a period (`.`) at the end of them, apart from the last item.

## Pull Request Checklist
When you submit your pull request, or you push new commits to it, automated systems might run some checks on your new code. Your pull request should pass these checks.

We also have more criteria than just that before we can
accept and merge it, usually a "does this help" test.
